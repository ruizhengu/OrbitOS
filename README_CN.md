# OrbitOS

[English](README.md) | 中文

> 一个**由 AI 驱动**的个人生产力系统，通过 AI 助手智能编排**知识管理**与**每日任务规划**。

![Screenshot](50_Resources/Screenshot.png)

## 什么是 OrbitOS？

OrbitOS 是一个基于 Obsidian 的生产力框架，其核心理念非常简单：**万物皆以你为中心旋转（Everything orbits around you）**。你的项目、知识和每日任务始终保持运动并相互连接——这一切都通过与 AI 的自然语言对话来管理。

与需要手动整理的传统笔记系统不同，OrbitOS 利用 **Claude Code** 或 **Gemini CLI** 作为你的智能知识管家和每日规划师。AI 不仅仅是存储信息，它还会主动帮助你：

- **捕获灵感**并将其转化为结构化、可落地的项目
- 根据你当前的活跃工作，提供具备上下文感知能力的**每日计划**建议
- **研究特定课题**并将发现自动组织到你的知识库中
- 通过智能双向链接（Wikilinks）**串联**笔记、项目与概念
- **归档与清理**已完成的工作，保持系统的精简与聚焦

## 核心特性

### AI 驱动的工作流

所有主要工作流都通过简单的斜杠命令（slash commands）启动，由 AI 承担繁重的工作：

| 工作流 | AI 的作用 |
| :--- | :--- |
| **每日规划** | 回顾昨日进展，浮现活跃项目，推荐重点领域，处理收件箱事项 |
| **项目创建** | 将粗略的想法转化为包含目标、阶段和成功指标的结构化项目 |
| **课题研究** | 执行深度调研，结构化调研结果，创建原子化维基条目，建立知识关联 |
| **知识解析** | 提取非结构化文本，并将其按适当的分类和链接整理到库（Vault）中 |
| **归档处理** | 识别已完成的项目并移至归档，同时保留历史背景 |

### 智能知识图谱

OrbitOS 广泛使用双向链接来构建连接的知识图谱：

- **项目 (Projects)** 链接到相关的 **研究 (Research)** 笔记，以获取背景和参考
- **每日笔记 (Daily notes)** 链接到你当天开展的项目
- **维基条目 (Wiki entries)** 是可以从任何地方引用的原子概念
- **研究笔记 (Research notes)** 综合信息并链接到源概念

AI 会在你的工作过程中自动创建这些连接，随时间推移构建起一套知识网。

### 结构化且灵活

文件夹结构提供了清晰的组织架构，同时保持了适应性：

```
├── 00_Inbox/          # 快速捕获 —— AI 会将其处理到合适的位置
├── 10_Daily/          # 每日日志 (YYYY-MM-DD.md) —— 每天早晨由 AI 生成
├── 20_Project/        # 活跃项目 —— AI 协助创建并跟踪进度
├── 30_Research/       # 深度研究笔记 —— 由 AI 根据你的探索进行结构化
├── 40_Wiki/           # 原子概念 —— AI 提取可复用的定义
├── 50_Resources/      # 精选内容 —— 通讯、产品发布、参考资料
├── 90_Plans/          # 执行方案 —— AI 起草，你批准，随后归档
└── 99_System/         # 系统配置
    ├── Archives/      # 历史记录 (按年/月组织)
    ├── Prompts/       # 针对不同领域的 AI 人设 (Personas)
    └── Templates/     # 保持一致性的 Markdown 模板
```

## 快速入门

### 前提条件

- 已安装 [Obsidian](https://obsidian.md/) 并打开此库 (Vault)
- 已配置 **Claude Code** 或 **Gemini CLI** 并在库目录下运行

### 你的第一天

1. **捕获第一个灵感**：在 `00_Inbox/` 中丢入一条快速笔记。
   - 只需写下想法、项目念头或想探索的东西。
   - 无需担心格式——AI 稍后会帮你进行结构化。

2. **开启新的一天**：在你的 AI 助手中运行 `/start-my-day`。
   - AI 会扫描收件箱并浮现需要处理的事项。
   - 回顾活跃项目（第一天是空的，这没关系！）。
   - 生成一份带有建议的每日笔记。

3. **启动项目**：运行 `/kickoff` 将收件箱项目转化为正式项目。
   - AI 会起草一份计划并询问澄清问题。
   - 创建一个包含目标、阶段和成功指标的结构化项目文件。

4. **研究课题**：运行 `/research <话题>`。
   - AI 执行彻底调查。
   - 在 `30_Research/` 中创建结构化笔记。
   - 将原子概念提取到 `40_Wiki/`。
   - 自动将所有内容链接在一起。

5. **快速提问**：运行 `/ask <问题>`。
   - 直接获取答案，无需创建沉重的文档。
   - 可选择将有用的发现保存到你的维基中。

## 命令指南

### 核心工作流

| 命令 | 用途 | 何时使用 |
| :--- | :--- | :--- |
| `/start-my-day` | AI 引导的每日规划与回顾 | 每天早晨用于设定焦点 |
| `/kickoff` | 将想法转化为结构化项目 | 启动任何新计划时 |
| `/research` | 带有自动知识结构化的深度调研 | 全面学习某项事物时 |
| `/ask` | 无需记录笔记的快速问答 | 简单问题、事实查询 |
| `/brainstorm` | 交互式灵感探索 | 开发和完善概念时 |
| `/parse-knowledge` | 将非结构化文本结构化到库中 | 处理笔记、文章、会议记录时 |
| `/archive` | 清理已完成的事项 | 定期维护、项目完成时 |

| `obsidian-bases` | 通过过滤器和公式创建类似数据库的视图 (.base 文件) |
| `json-canvas` | 视觉思维导图和流程图 (.canvas 文件) |

### 内容筛选（可选）

| :--- | :--- |

## 项目理念
## C.A.P. 项目布局

每个项目都遵循一致的结构：

- **Context (背景)**：我们要实现什么？成功的标准是什么？
- **Actions (行动)**：分阶段的任务清单。
- **Progress (进展)**：带有时间戳的更新，并链接到每日笔记。

### 双向链接与连接

OrbitOS 深度依赖 Obsidian 的双向链接语法：

```markdown
[[笔记名称]]                # 基础链接
[[笔记名称|显示文本]]       # 自定义显示文本
![[笔记名称]]               # 嵌入整篇笔记
![[笔记名称#章节]]          # 嵌入特定章节
```

AI 会自动创建这些链接，但你也应该积极地进行手动链接——连接产生价值。

### 以每日笔记为锚点

每日笔记 (`10_Daily/YYYY-MM-DD.md`) 是你的核心锚点：

- 每一个项目更新都会引用开展该工作的每日笔记。
- 收件箱事项被处理后，会从每日笔记中链接出去。
- `/start-my-day` 工作流会自动生成这些笔记。

## 设计哲学

OrbitOS 基于以下原则构建：

1. **AI 即伙伴**：AI 不仅仅是一个工具，它是一个理解你的系统并协助维护的积极协作伙伴。
2. **全量捕获，后续处理**：收件箱的存在是为了让你永不丢失灵感；AI 会在你准备好时协助处理。
3. **连接优于分类**：僵化的文件夹层级终会失效；双向链接创造了一个灵活、可查询的知识图谱。
4. **每日节奏**：每日笔记锚定一切，为你的工作和思考建立时间线。
5. **渐进式形式化**：想法始于粗略，通过 AI 辅助的精炼逐渐变得结构化。
